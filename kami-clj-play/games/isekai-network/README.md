# isekai-network — Ghost Hacker: Shiro & Pico

Game project codename for **Ghost Hacker: Shiro & Pico**, a network-isekai
spin-off built on the CLJ game layer (ADR-0035/0036/0037/0038). Franchise
canon (series bible, character/monster design spec) lives in
`orgs/gftdcojp/ai-gftd-mangaka/data/ghosthacker-shiropico/` — this directory
only maps that canon onto playable mechanics.

## What's actually implemented here

Only **concept 1 (熱痕ランナー)** has running code, and it is an intentionally
faithful rename+recolour of `kami-clj-play/games/survivors` — same systems,
same world tuning, same compiler subset. This is a scoped first slice, not a
claim that all 5 concepts are implemented.

- `author.clj` — datalevin (Datalog) owns the render profiles (`shiro-pico`,
  `ghost`) and world/fx tuning; transacts → queries → projects `scene.edn`.
  Colours are taken from the canonical palette in
  `character-design-spec.json` (shiro teal `#1FB6A6`, ghost body `#29D3C2`,
  "heat afterimage" burst `#F4F8F7`).
- `logic.clj` — gameplay in the kami-clj subset: `control`/`spawn`/`ai`/
  `weapon`/`contact` systems. `weapon` = シロの一撃 (ranged, periodic,
  visualize-and-purify nearest ghost). `contact` = ピコの刈り取り (purify on
  touch). Both currently read a single `"shiro-pico"` entity — the duo moves
  as one controllable unit in this mode (see `design/01-netsurvivors.edn`
  `:concept/genre-note`).
- `scene.edn` — generated by `author.clj`. Do not hand-edit.

### Build / run

```bash
# 1. author the scene (regenerate scene.edn from datalevin)
cd kami-clj-play/games/isekai-network
clojure -M author.clj

# 2. compile the gameplay logic to WASM
cd ../../..   # repo root
cargo run -p kami-engine-clj --bin kamiclj -- \
  kami-clj-play/games/isekai-network/logic.clj -o /tmp/isekai-network.wasm
```

Not yet wired: an actual `kami-clj-play` binary invocation that loads this
game directory end-to-end in a window (mirrors how `survivors`/`waves` are
driven — see their READMEs / `kami-script-runtime` for the host-side loader).

## Not yet done (honest scope)

- Concepts 2–5 are design-only; no `logic.clj`/`author.clj` for them yet.
- Daemon/Zombie bestiary members are not wired into concept 1's `logic.clj`
  (only `ghost`) — multi-tag enemy spawning in the kami-clj subset is
  unverified, flagged in `design/01-netsurvivors.edn`.
- No music/audio layer implemented for any concept. The FreeTEMPO-style
  lounge/house direction described in `design/00-game-bible.edn`
  `:shared-foundation :engine-note` requires a beat-grid-driven generative
  music layer that does not exist yet — `kami-live`'s `:dance/*` beat grid is
  flagged as the most promising reuse target, not a working integration.
